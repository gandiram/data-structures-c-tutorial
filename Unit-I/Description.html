Data Structures Overview

1. Structures

Structures are a way to group different data types together. They are useful for managing complex data.

struct Student {
    char name[50];
    int age;
    float gpa;
};

2. Unions

Unions allow storing different data types in the same memory location. Only one value can be stored at a time.

union Data {
    int intValue;
    float floatValue;
    char charValue;
};

3. Typedef

Typedef provides a way to create aliases for existing data types.

typedef unsigned long ulong;
ulong myVar;

4. Enumerations

Enumerations define a variable that can be assigned a set of predefined constant values.

enum Day { Sunday, Monday, Tuesday, Wednesday, Thursday, Friday, Saturday };
enum Day today = Monday;

Quick Reference

  • Structures: Group data types.
  • Unions: Save memory, only one type at a time.
  • Typedef: Create aliases for data types.
  • Enumerations: Set of named integer constants.

Best Practices

Always ensure clear naming conventions for your structures, unions, and enums to maintain code readability.
