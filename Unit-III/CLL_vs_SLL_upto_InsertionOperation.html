What is SLL?

Nodes linked in one direction. Last node's next = NULL. Traversal stops at NULL — it's a dead end.

What is CLL?

Last node's next = head. No NULL. Forms a continuous circle. Must use do-while(curr != head) for traversal.

🔗 SLL Insert Begin

SLL Insert at Beginning

O(1) — Just update the head pointer, no traversal needed

Insert at Beginning — O(1)

1

2
// Step 1: Allocate new node
3

4
// Step 2: new→next = old head
5

6
// Step 3: head = new node
7
*head = n;
8
} // Time: O(1) — no traversal needed!

Why O(1)?

We never loop through the list. Just two pointer assignments: n→next = head and head = n. Constant time regardless of list size.

🔗 SLL Insert End

SLL Insert at End

O(n) — Must traverse to find the last node (NULL check)

Insert at End — O(n)
sll_insertEnd.c
1
void insertEnd(struct Node **head, int data) {
2
struct Node *n = createNode(data);
3
if(*head == NULL){ *head = n; return; }
4
// traverse to find last node
5
struct Node *curr = *head;
6
while(curr->next != NULL)
7
curr = curr->next; // find last
8
curr->next = n; // link last → new
9
} // Time: O(n) — must find last node

Why O(n)?

We must walk the entire list until curr→next == NULL to find the last node. For a list of n nodes, this takes n steps — O(n) time.

🔗 SLL Insert Middle

SLL Insert at Middle

O(n) — Traverse to position, then do A before B!

Insert at Middle (after given position) — O(n)
sll_insertMiddle.c
1
void insertAt(struct Node **head, int data, int pos) {
2
struct Node *n = createNode(data);
3
struct Node *curr = *head;
4
// pos >= 2: traverse to node at (pos-1)
5
for(int i=1; i < pos-1; i++)
6
curr = curr->next; // reach predecessor
7
// ⚠️ ORDER MATTERS! A before B
8
n->next = curr->next; // Step A
9
curr->next = n; // Step B
10
} // Time: O(n)
▶ Insert 25 after pos 2 in 10→20→30→NULL
Starting list: 10→20→30→NULL. Insert 25 after position 2 (after node 20).
0/6

⚠️ Critical: Order Matters!

Always do Step A first: n→next = curr→next (save connection to rest of list), then Step B: curr→next = n. Reversing the order permanently loses the rest of the list!

🔄 CLL Insert Begin

CLL Insert at Beginning

O(n) — Must find the last node to maintain the circular link

CLL Insert at Beginning — O(n)
cll_insertBegin.c
1
void cllInsertBegin(struct Node **head, int data) {
2
struct Node *n = createNode(data);
3
n->next = n; // points to itself initially
4
if(*head == NULL){ *head = n; return; }
5
// Find the LAST node (it points to head)
6
struct Node *last = *head;
7
while(last->next != *head)
8
last = last->next;
9
// Insert: new→next = old head
10
n->next = *head;
11
// last→next = new (maintains circle)
12
last->next = n;
13
// Update head
14
*head = n;
15
} // Time: O(n) — must update last node!
▶ Insert 5 at beginning of CLL: 10→20→30→↩HEAD
CLL: 10→20→30→↩HEAD. Insert 5 at beginning.
0/7

⚠️ CLL Insert Begin is O(n) — unlike SLL's O(1)!

In SLL, inserting at the beginning is O(1) because we just update the head. But in CLL, the last node must always point to the current head. When head changes, we must traverse to find the last node and update its next pointer — that's O(n).

🔄 CLL Insert End

CLL Insert at End

O(n) — Traverse to find last, then set n→next = head (not NULL!)

CLL Insert at End — O(n)
cll_insertEnd.c
1
void cllInsertEnd(struct Node **head, int data) {
2
struct Node *n = createNode(data);
3
n->next = n;
4
if(*head == NULL){ *head = n; return; }
5
// Traverse to find last node
6
struct Node *last = *head;
7
while(last->next != *head)
8
last = last->next;
9
// Link: last → new → head
10
last->next = n;
11
n->next = *head; // new node → back to head
12
} // Time: O(n)
▶ Insert 40 at end of CLL: 10→20→30→↩HEAD
CLL: 10→20→30→↩HEAD. Insert 40 at end.
0/6

Key Difference from SLL Insert End

In SLL we set curr→next = n (n→next is already NULL). In CLL we must also set n→next = head to close the circle. Never leave it as NULL!

Key Takeaways

⚠️ CLL Insert Begin = O(n)

Unlike SLL's O(1), CLL must find the last node to update its next pointer whenever the head changes. This traversal costs O(n).

✅ CLL Insert Middle = Same as SLL

Circular property is preserved automatically since only middle pointers are re-linked. The last node's connection to head is never disturbed.

🚫 Never use NULL check in CLL

In CLL, no node ever points to NULL. Always use curr != head as the loop termination condition.
