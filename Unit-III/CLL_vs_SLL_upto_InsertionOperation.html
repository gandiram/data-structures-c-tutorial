🔄 CLL vs SLL — Key Difference

CLL Concept

SLL: last→next = NULL (dead end). CLL: last→next = head (circle!). Never use curr==NULL in CLL!

SLL vs CLL: The Key Difference

What makes CLL different from SLL?

In a Singly Linked List (SLL), the last node's next pointer is NULL — a dead end. In a Circular Linked List (CLL), the last node's next points back to head, forming a circle. This means traversal never naturally hits NULL — you must use do-while(curr != head) to stop.

Key Structural Differences

SLL Structure

HEAD → [10|→] → [20|→] → [30|NULL]

Dead end at NULL. The last node's next pointer is NULL. Traversal naturally stops when curr == NULL.

CLL Structure

HEAD → [10|→] → [20|→] → [30|→HEAD]

Wraps back to head. The last node's next points back to head. Forms a complete circle — no NULL sentinel.

⚠️ Critical Rule

NEVER use curr==NULL to stop traversal in CLL! It will loop forever. Use:

do { ... } while(curr != head)

Why CLL Insert Begin is O(n) but SLL Insert Begin is O(1)

The O(n) vs O(1) Insert-Begin Puzzle

In SLL, head is given directly, so we just prepend: set n→next = head, then head = n. No traversal needed — pure O(1).

In CLL, the circle must be maintained — the last node's next must be updated to point to the new head. To find the last node, we must traverse the entire list: while(last→next != head). This traversal makes it O(n).

💡 Tip: If we maintained a tail pointer, CLL insert-begin could also be O(1)!
