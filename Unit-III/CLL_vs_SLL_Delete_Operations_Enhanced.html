1. Delete Operation Basics

Deleting a node means removing that node from the list and then freeing its memory. The main idea is simple, but the pointer updates are different in CLL and SLL.

SLL End Rule

The last node points to NULL.

CLL End Rule

The last node points back to HEAD.

Delete Impact

In CLL, you often need one extra relink so the circle remains closed.

Main difference: In SLL, deletion usually reconnects the chain toward NULL. In CLL, deletion must preserve the circular connection back to HEAD.

2. Source-Code Observations

The new PDF is based on the actual delete functions found in the attached C files.

Important source behavior: The function deleteVal(int key) in both files works like "delete by value in middle / non-head deletion". In the attached source code, deleting the head node by value is not handled separately. This document therefore explains delete in middle using a middle node example, which matches the code behavior more safely.

CLL source observations

  • deleteBegin() finds the last node and reconnects it to the new head.
  • deleteEnd() uses the second-last node to remove the last node.
  • deleteVal() uses prev and curr inside a circular traversal.

SLL source observations

  • deleteBegin() simply moves head to the next node.
  • deleteEnd() uses prev and curr to unlink the tail.
  • deleteVal() also uses prev and curr to bypass the target node.

deleteBegin()
CLL - cll_new.c SLL - sll.c 
void deleteBegin() {
    if (head == NULL) {
        printf("CLL is Empty\n");
        return;
    }
    if (head->next == head) {
        head = NULL;
        return;
    }
    struct Node *last = head;
    while (last->next != head)
        last = last->next;
    last->next = head->next;
    struct Node *temp = head;
    head = head->next;
    free(temp);
}
 
void deleteBegin() {
    if (head == NULL) {
        printf("Empty list\n");
        return;
    }
    struct Node *temp = head;
    head = head->next;
    free(temp);
}
Main Difference
CLL must reconnect the last node to the new head so the circle is preserved. SLL only moves the head forward and frees the old head.
deleteVal(int key) - delete in middle
CLL - cll_new.c SLL - sll.c 
void deleteVal(int key) {
    struct Node *curr = head, *prev = NULL;
    do {
        if (curr->data == key)
            break;
        prev = curr;
        curr = curr->next;
    } while (curr != head);

    if (curr == head && curr->data != key) {
        printf("not found\n");
        return;
    }
    prev->next = curr->next;
    free(curr);
}
 
void deleteVal(int key) {
    struct Node *curr = head, *prev = NULL;
    while (curr != NULL && curr->data != key) {
        prev = curr;
        curr = curr->next;
    }
    if (curr == NULL) {
        printf("Not found");
        return;
    }
    prev->next = curr->next;
    free(curr);
}
Main Difference
The bypass step is similar in both lists. The search loop differs because CLL searches in a circle and must detect when it returns to HEAD, while SLL stops at NULL.
deleteEnd()
CLL - cll_new.c SLL - sll.c 
void deleteEnd() {
    if (head == NULL) {
        printf("CLL is empty\n");
        return;
    }
    if (head->next == head) {
        free(head);
        head = NULL;
        return;
    }
    struct Node *sec_last = head;
    while (sec_last->next->next != head)
        sec_last = sec_last->next;
    struct Node *temp = sec_last->next;
    sec_last->next = temp->next;
    free(temp);
}
 
void deleteEnd() {
    if (head == NULL)
        return;
    if (head->next == NULL) {
        free(head);
        head = NULL;
        return;
    }
    struct Node *prev = NULL, *curr = head;
    while (curr->next != NULL) {
        prev = curr;
        curr = curr->next;
    }
    prev->next = NULL;
    free(curr);
}
Main Difference
CLL uses the second-last node and reconnects it to HEAD. SLL uses prev and curr, then makes the new tail point to NULL.
Operation Purpose and Key Rule Complexity
deleteBegin() Delete the first node. CLL must reconnect the last node to the new head. SLL only advances head. CLL: O(n)
SLL: O(1)
deleteVal() Delete a target value from the middle. Both bypass the target using prev and curr. CLL loops until HEAD, SLL loops until NULL. CLL: O(n)
SLL: O(n)
deleteEnd() Delete the last node. CLL reconnects the second-last node to HEAD. SLL reconnects it to NULL. CLL: O(n)
SLL: O(n)

7. Quick Summary

What to remember

  • CLL deletion must preserve the circular link.
  • SLL deletion preserves a straight chain ending in NULL.
  • deleteBegin is simpler in SLL than CLL.
  • deleteEnd needs traversal in both lists.
  • deleteVal uses bypass logic in both lists.

Complexity recap

  • deleteBegin: CLL O(n), SLL O(1)
  • deleteVal: CLL O(n), SLL O(n)
  • deleteEnd: CLL O(n), SLL O(n)
